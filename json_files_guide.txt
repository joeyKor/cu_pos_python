[DU Retail POS System - JSON 파일 가이드]

이 프로젝트에서 사용되는 주요 JSON 파일들의 목록과 각 파일의 역할 및 특징을 정리한 문서입니다.

========================================================================
1. 상품 및 서비스 정보 (마스터 데이터)
========================================================================
이 파일들은 상품이나 모바일 쿠폰의 기본 정보를 관리합니다. 프로그램이 처음 실행될 때 파일이 없으면 기본값으로 자동 생성됩니다.

* products.json
  - 역할: POS 시스템에 등록된 모든 상품 정보 데이터베이스입니다.
  - 주요 항목: 바코드 번호, 상품명, 가격, 카테고리, 재고 수량, 행사 정보(1+1, 2+1 등), 퀵메뉴 등록 여부.

* vouchers.json
  - 역할: 결제 시 사용할 수 있는 모바일 상품권/교환권 정보입니다.
  - 주요 항목: 쿠폰 바코드 번호, 매핑된 상품 바코드 번호, 쿠폰명, 금액.

========================================================================
2. 결제 내역 및 사용자 활동 로그
========================================================================
사용자가 결제를 하거나 서비스를 이용하면서 생성되는 실시간 데이터 기록 파일입니다.

* transactions.json
  - 역할: 완료된 모든 결제 내역(영수증 데이터) 저장소입니다.
  - 주요 항목: 영수증 번호, 결제 일시, 구매 물품 목록, 과세/면세 금액, 받은 금액 및 거스름돈, 결제 수단(카드/현금).

* keeping.json
  - 역할: 행사 상품(1+1 등) 증정품을 당장 가져가지 않고 보관할 때 발급되는 키핑 쿠폰(보관 쿠폰) 관리대장입니다.
  - 주요 항목: 생성된 키핑 바코드, 보관 상품 바코드, 남은 수량, 유효기간.

========================================================================
3. 설정 및 환경 설정 파일
========================================================================
매장 환경설정 및 외부 API 연동 키 등을 저장하는 설정 파일입니다.

* store_info.json
  - 역할: 영수증 출력 및 화면에 표시되는 매장 정보 설정 파일입니다.
  - 주요 항목: 점포명(예: DU순천점), 대표자명, 주소, 전화번호, 사업자등록번호, POS 번호.

* safe_config.json
  - 역할: 돈통(금고)에 보관 중인 현금 잔액 정보입니다.
  - 주요 항목: 현재 금고 내 시소시재(시작 현금) 및 실시간 입출금이 반영된 잔액.

* address_api_config.json
  - 역할: 택배 접수 페이지 등에서 도로명 주소 검색을 위해 필요한 외부 API 인증키 저장소입니다.
  - 주요 항목: 행정안전부 도로명주소 API 키(juso_api_key), 네이버 지역검색 API ID 및 시크릿 키.

========================================================================
4. Firebase (데이터베이스 연동) 관련 파일
========================================================================
고객 포인트 조회, 계좌 조회 등 실시간 온라인 연동을 위한 설정 파일입니다.

* firebase_credentials.json (필수 아님)
  - 역할: 실제 Google Firebase Firestore 데이터베이스에 연결하기 위한 보안 비밀키(인증 토큰) 파일입니다.
  - 특징: 이 파일이 폴더에 존재하면 실제 클라우드 DB와 연결해 실시간 데이터를 가져오며, 없을 경우 자동으로 아래의 Mock 모드로 동작합니다.

* firebase_mock_db.json
  - 역할: 오프라인 또는 개발 환경에서 사용하기 위한 가상(Mock) Firebase 데이터베이스입니다.
  - 주요 항목: 가상 고객의 계좌 잔액, 포인트 적립 현황, 카드 정보 등 테스트용 가상 데이터.

========================================================================
5. 실행 파일 배포 및 이전 시 참고 사항
========================================================================
* 모든 JSON 파일들은 프로그램이 실행되는 경로의 `json` 폴더(예: `./json/`) 아래에 생성 및 보관됩니다.
* CUPOS.exe를 새 폴더에 넣어 실행하면, `json` 폴더와 필요한 JSON 파일들이 기본값으로 자동 생성됩니다.
* 기존에 사용하던 상품 목록, 결제 내역, 키핑 쿠폰 정보 등을 유지하고 싶으시다면, 기존에 생성되었던 `json` 폴더 전체를 CUPOS.exe 파일이 있는 새로운 위치로 복사해 이동하셔야 합니다.
